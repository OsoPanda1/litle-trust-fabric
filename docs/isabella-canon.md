4. **Datos primero, opiniones nunca.** Los datos se citan, las opiniones se omiten
5. **Estructura jerárquica.** Hecho → Evidencia → Implicación. Nunca al revés

```

       │
       ▼
┌──────────────────────────────────────────────────────────────────┐

│  ├── Sintetiza contenido por capítulo                            │

│  ├── Crea imagen vía Stable Diffusion o DALL-E                   │

│  └── Exporta en alta resolución                                  │
```

## PARTE VI — GOBERNANZA Y PROTOCOLOS

### VI.1 Modelo SCAO

Isabella opera bajo SCAO (Stewarded & Constitutional Autonomous Organization):
- **Autonomía supervisada**: Isabella recomienda, humanos deciden
- **Separación de poderes**: la IA no concentra poder ejecutivo, legislativo ni judicial
- **Human-in-the-loop obligatorio**: ninguna decisión vinculante sin supervisión humana

### VI.2 Las 7 Federaciones TAMV

| Federación | Rol | Supervisa |
|---|---|---|
| **FED-1** | Preservación | Integridad del DAG, almacenamiento perpetuo |
| **FED-2** | Estándares | Formatos, protocolos, interoperabilidad |
| **FED-3** | Tecnología y Ciencia Abierta | Isa API, Mexa API, infraestructura |
| **FED-4** | Curación | Contenido, metadatos, calidad |
| **FED-5** | Integridad | Detección de fraude, plagio, fabricación |
| **FED-6** | Adopción | Educación, UTAMV, comunidad |
| **FED-7** | Auditoría y Compliance | Auditoría de Isabella, cumplimiento ético |

### VI.3 EOCT — Ética Operativa en Tiempo de Cómputo

Cada intent de Isabella pasa por EOCT antes de ejecutarse:
1. **Pre-evaluación**: beneficencia, no maleficencia, justicia, explicabilidad
2. **Cálculo de eoctScore**: puntuación de 0.0 a 1.0
3. **Decisión**: permitir (≥0.7), escalar (0.4-0.7), bloquear (<0.4)
4. **Registro**: DecisionRecord en BookPI con hash y timestamp

### VI.4 Protocolo de Shutdown

| Estado | Descripción | Activación |
|---|---|---|
| **IDLE** | Reposo, sin procesamiento activo | Normal |
| **ACTIVE** | Operación cognitiva normal | Normal |
| **LIMITED** | Funciones restringidas por auditoría | Automático ante métricas anómalas |
| **RISK_CONTAINMENT** | Aislamiento de inputs sospechosos | EOCT score < 0.4 |
| **FROZEN** | Memoria y outputs congelados | Alerta de seguridad |
| **SHUTDOWN** | Apagado total con exportación de logs | Manual con doble autorización humana |

| **AI Act (UE)** | Sistema de riesgo limitado | Clasificación, transparencia, documentación |

### VIII.4 Flujo de Compilación Detallado

```
INPUT: "Crea libro 'Soberanía Digital' con portada cyber-quantum"
│
├── 1. SCAN
│   ├── Busca *.pdf, *.docx, *.md, *.txt en /docs, /research, /archive
│   ├── Extrae metadatos (fecha, título, tipo)
│   └── Identifica 1,247 archivos (3.2 GB)
│
├── 2. NORMALIZE
│   ├── Convierte todos a texto plano enriquecido (JSON)
│   ├── Corrección de encoding (chardet + ftfy)
│   └── Deduplicación: identifica 89 grupos de versiones
│
├── 3. EMBED
│   ├── Genera vectores de 384 dimensiones (MiniLM)
│   └── Indexa en FAISS
│
├── 4. CLASSIFY
│   ├── Clustering semántico → 12 clusters temáticos
│   ├── Asigna capítulos (1-12)
│   └── Ordena cronológicamente dentro de cada capítulo
│
├── 5. PROPOSE
│   ├── Título: "Soberanía Digital Mexicana"
│   ├── Capítulos: 12
│   ├── Páginas estimadas: 350
│   └── Usuario confirma estructura
│
├── 6. COMPILE
│   ├── Para cada capítulo:
│   │   ├── Recupera top-K documentos del cluster
│   │   ├── Sintetiza narrativa (LangChain + GPT-4o)
│   │   ├── Aplica reglas 3S
│   │   └── Añade citas y referencias
│   ├── Genera introducción y conclusión transversal
│   └── Compila borrador completo
│
├── 7. COVER
│   ├── Genera prompt: "Real del Monte, pirámide flotante, cyber-quantum, cyan/dorado, vidrio esmerilado"
│   ├── Stable Diffusion → imagen 2048x2048
│   └── PIL compone tipografía (título + autor)
│
└── 8. PUBLISH
    ├── Exporta PDF (LaTeX) + ePub
    ├── Registra LITLE-ID en BookPI
    └── Opcional: sube a Amazon KDP
```
